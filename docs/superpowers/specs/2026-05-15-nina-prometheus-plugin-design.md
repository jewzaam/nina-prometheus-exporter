# NINA Prometheus Exporter Plugin — Design (v0.1.0)

Companion to `nina-prometheus-plugin-spec.md`. The spec defines **what** to expose; this document defines **how** to build it. Reading order: spec first, then this design.

License: Apache-2.0. Initial version: `0.1.0`. Target NINA: 3.x (MinimumApplicationVersion pinned to current `NINA.Plugin` NuGet version at publish time).

## 1. Decisions Locked During Brainstorming

## 3. Build Infrastructure (Makefile)

Makefile is the project's CLI interface. Every recurring developer action goes through a target — no ad-hoc shell invocations.

Targets follow the **verb-[qualifier-]noun** grammar from `~/source/standards/build/makefile.md`. Whitelisted bare verbs: `help`, `check`, `clean`, `format`, `install`, `uninstall`, `run`. Everything else carries a verb prefix. Default target is `check` (read-only quality gate; CI also calls `check`).

| Target | Purpose |
|---|---|
| `help` | Self-documenting target list (greps `##` comments). |
| `check` | Aggregate read-only gate: `test-env` → `build-release` → `test-unit` → `test-format`. Default target. |
| `clean` | `dotnet clean` + remove `bin/`, `obj/`. |
| `format` | `dotnet format` (rewrites sources in place). |
| `install` | Composite: `build-release` → copy `bin/Release/net8.0/*.dll` (plugin + non-NINA deps) into `%LOCALAPPDATA%\NINA\Plugins\Prometheus Exporter\`. |
| `install-dev` | Composite: `install` → `run-nina` (dev-loop one-shot). |
| `uninstall` | Remove the plugin install directory. |
| `run-nina` | Kill any running `NINA.exe`, relaunch NINA. Used for plugin reload during development. |
| `build-debug` | `dotnet build -c Debug`. |
| `build-release` | `dotnet build -c Release`. |
| `build-package` | Zip Release output as `nina-prometheus-exporter-<version>.zip` for GitHub release. |
| `test-env` | Preflight: verify .NET 8 SDK, NINA install path, `%LOCALAPPDATA%\NINA\Plugins\` writable. Print versions. Exit non-zero on missing prerequisite. |
| `test-unit` | `dotnet test` (xUnit). |
| `test-format` | `dotnet format --verify-no-changes` (read-only; exits non-zero if any file needs reformat). |

Runs under GNU make for Windows (chocolatey: `choco install make`). Targets use PowerShell via `pwsh -NoProfile -Command` so behavior matches the developer shell.

`install` is project-mutated (composite) per the standard: in the .NET-plugin context, "install everything for end-user use" = build Release + copy DLLs to NINA's plugin directory. `install-dev` adds the NINA relaunch on top for plugin-reload during local iteration.

## 4. Plugin Skeleton + MEF Wiring

```csharp
[Export(typeof(IPluginManifest))]
public sealed class PrometheusExporterPlugin : PluginBase {
    private readonly IProfileService _profileService;
    private readonly EquipmentMetrics _equipment;
    private readonly CameraMetrics _camera;
    private readonly MountMetrics _mount;
    private readonly FocuserMetrics _focuser;
    private readonly FilterWheelMetrics _filter;
    private readonly GuiderMetrics _guider;
    private readonly ImageMetrics _image;
    private readonly AutoFocusMetrics _autofocus;
    private readonly SequenceMetrics _sequence;
    private readonly DomeMetrics _dome;
    private readonly RotatorMetrics _rotator;
    private readonly FlatDeviceMetrics _flat;
    private readonly SafetyMetrics _safety;
    private readonly WeatherMetrics _weather;
    private readonly PrometheusServer _server;
    private readonly PrometheusExporterOptions _options;

    [ImportingConstructor]
    public PrometheusExporterPlugin(
        IProfileService profileService,
        ICameraMediator cameraMediator,
        ITelescopeMediator telescopeMediator,
        IFocuserMediator focuserMediator,
        IFilterWheelMediator filterWheelMediator,
        IGuiderMediator guiderMediator,
        IImageSaveMediator imageSaveMediator,
        ISequenceMediator sequenceMediator,
        IDomeMediator domeMediator,
        IRotatorMediator rotatorMediator,
        IFlatDeviceMediator flatDeviceMediator,
        ISafetyMonitorMediator safetyMonitorMediator,
        IWeatherDataMediator weatherMediator,
        IImagingMediator imagingMediator) {
        _profileService = profileService;
        _options = new PrometheusExporterOptions(this);
        // Construct collectors (no event subscriptions yet)
        _equipment = new EquipmentMetrics(cameraMediator, telescopeMediator, focuserMediator,
            filterWheelMediator, guiderMediator, domeMediator, rotatorMediator,
            flatDeviceMediator, safetyMonitorMediator, weatherMediator);
        _camera = new CameraMetrics(cameraMediator);
        _mount = new MountMetrics(telescopeMediator);
        _focuser = new FocuserMetrics(focuserMediator);
        _filter = new FilterWheelMetrics(filterWheelMediator);
        _guider = new GuiderMetrics(guiderMediator);
        _image = new ImageMetrics(imageSaveMediator);
        _autofocus = new AutoFocusMetrics(focuserMediator, _options);
        _sequence = new SequenceMetrics(sequenceMediator, _options);
        _dome = new DomeMetrics(domeMediator);
        _rotator = new RotatorMetrics(rotatorMediator);
        _flat = new FlatDeviceMetrics(flatDeviceMediator);
        _safety = new SafetyMetrics(safetyMonitorMediator);
        _weather = new WeatherMetrics(weatherMediator);
        _server = new PrometheusServer();
        _options.PropertyChanged += OnOptionsChanged;
    }

    public override Task Initialize() {
        SetDefaultLabels();
        foreach (var c in AllCollectors()) c.Subscribe();
        _sequence.StartPolling();
        _server.Start(_options.BindAddress, _options.Port);   // always on
        return Task.CompletedTask;
    }

    public override Task Teardown() {
        _server.Stop();
        _sequence.StopPolling();
        foreach (var c in AllCollectors()) c.Dispose();
        _options.PropertyChanged -= OnOptionsChanged;
        return Task.CompletedTask;
    }
}
```

Constructor stays side-effect-free (no I/O, no event wiring). `Initialize()` runs after every plugin has been loaded, which is the documented requirement for `ISequenceMediator` use.

## 5. AssemblyInfo

Cribbed from `nina-influxdb-exporter/Properties/AssemblyInfo.cs` and `nina.plugin.template`. Generate the GUID **once** and never change it (rename breaks user sequence-saved type references).

The published/human-facing version is **SemVer 3-segment** (`0.1.0`) per `~/source/standards/common/versioning.md` — README, CHANGELOG, GitHub release tag, manifest entry. `[AssemblyVersion]` is also 3-segment (used only for .NET binding). **`[AssemblyFileVersion]` must be 4-segment**: NINA's `PluginLoader.LoadPlugin` constructs a `System.Version` from `AssemblyFileVersion` and rejects negative revisions, so `"0.1.0"` parses as `0.1.0.-1` and fails. The fixed trailing `.0` build segment is invisible to users.

## 6. Options (4 Fields, IPluginOptionsAccessor-Backed)

Server is **always on** while the plugin is loaded — no enable/disable toggle. To stop the exporter, uninstall the plugin. All four fields are exposed in the Options UI (`Options.xaml`) and persisted via `IPluginOptionsAccessor`.

| Field | Type | Default | Bounds | Notes |
|---|---|---|---|---|
| `Port` | int | `9876` | 1–65535 | 9090-range, not in [common exporter port allocations](https://github.com/prometheus/prometheus/wiki/Default-port-allocations). Pre-publish: cross-check final allocation. |
| `BindAddress` | string | `0.0.0.0` | IP or hostname | Configurable in UI. Default exposes on LAN to match current scrape topology. README documents the security implication. |
| `SequencePollIntervalSeconds` | int | `1` | 1–60 | Sequence-status poll cadence. UI label: "Sequence poll interval (seconds)". |
| `AfTimeoutMinutes` | int | `10` | 0–120 | 0 disables timeout. UI label: "Autofocus timeout (minutes)". |

**Apply-without-restart handler** (`OnOptionsChanged`):

| Property changed | Action |
|---|---|
| `Port` or `BindAddress` | `_server.Restart(BindAddress, Port)` — stop `HttpListener`, start new one. Metric registry untouched. |
| `SequencePollIntervalSeconds` | `_sequence.RestartTimer(newSeconds)` |
| `AfTimeoutMinutes` | Stored in options; next AF run picks it up. Active timeout unchanged. |

**Failure modes**: `MetricServer.Start()` throws if port is in use or bind address invalid. Catch in `PrometheusServer.Start()`, log via NINA's `Logger`, set internal `Status` to `"stopped — port in use"` and expose in Options UI. **Do not crash NINA.**

## 7. HTTP Server (`PrometheusServer.cs`)

**Backend: EmbedIO (`HttpListenerMode.EmbedIO`), not prometheus-net's `MetricServer`.**

prometheus-net's bundled `MetricServer` wraps `System.Net.HttpListener`, which routes through Windows' `http.sys` driver. `http.sys` reserves every non-loopback URL prefix for admin — an unprivileged process (NINA) gets `Access is denied` (HttpListenerException error code 5) when binding to `0.0.0.0`, `+`, or any specific LAN IP. The only workaround is a one-time elevated `netsh http add urlacl ...` grant, which is unacceptable UX for a NINA plugin (users don't have a Makefile, NINA can't self-elevate).

`EmbedIO` (NuGet `EmbedIO 3.5.2`, pattern lifted from `~/source/ninaAPI/ninaAPI/WebService/API.cs`) ships its own pure-managed socket listener. Using `HttpListenerMode.EmbedIO` bypasses `http.sys` entirely, so binding LAN-exposed addresses works from a normal user process with no setup.

We keep `prometheus-net` for metric instances (gauges/counters/labels) and the text-format serializer. EmbedIO is the HTTP transport only.

Implementation:

```csharp
internal sealed class PrometheusServer {
    private WebServer _server;
    private CancellationTokenSource _cts;
    private Thread _serverThread;
    public string Status { get; private set; } = "stopped";

    public void Start(string bindAddress, int port) {
        var listenerHost = (bindAddress == "0.0.0.0" || bindAddress == "+" ||
                            string.IsNullOrWhiteSpace(bindAddress)) ? "*" : bindAddress;
        var prefix = $"http://{listenerHost}:{port}/";

        _cts = new CancellationTokenSource();
        _server = new WebServer(o => o
            .WithUrlPrefix(prefix)
            .WithMode(HttpListenerMode.EmbedIO))
            .WithAction("/metrics", HttpVerbs.Get, async ctx => {
                using var ms = new MemoryStream();
                await Metrics.DefaultRegistry.CollectAndExportAsTextAsync(ms, ctx.CancellationToken);
                await ctx.SendStringAsync(Encoding.UTF8.GetString(ms.ToArray()),
                                          "text/plain", Encoding.UTF8);
            });

        _serverThread = new Thread(() => _server.RunAsync(_cts.Token).GetAwaiter().GetResult())
            { Name = "PrometheusExporter HTTP", IsBackground = true };
        _serverThread.Start();
        Status = $"running on {bindAddress}:{port}";
    }

    public void Stop() { _cts?.Cancel(); _server?.Dispose(); /* ... */ }
    public void Restart(string b, int p) { Stop(); Start(b, p); }
}
```

**Default labels** — `profile_name` and `host_name` attached to every metric so multi-NINA-instance scrapes stay distinguishable. Implementation: each collector receives these in its ctor and applies via `.WithLabels(profile, host)` when declaring metric instances, or via a shared `Metrics.WithLabels(...)` factory on `Metrics.DefaultRegistry`. Choice deferred to implementation — both work, factory-pattern is less boilerplate. Profile name re-read on each NINA profile switch via `IProfileService` event.

**Naming convention**: `nina_<family>_<metric>_<unit>`. Unit suffix in metric name follows Prometheus convention (`_celsius`, `_arcsec`, `_percent`, `_seconds`, `_bytes`). Bool gauges 0/1 — no unit suffix. Counters end in `_total`.

## 9. Sequence-Status Polling (`SequenceMetrics.cs`)

```
state: running_set = {}  // set of (category, item) tuples
timer fires every options.SequencePollIntervalSeconds:
  try:
    current = sequenceMediator.GetAdvancedSequencerCurrentRunningItems()
      → returns IList<IList<ISequenceItem>>; flatten + derive (category, item) tuples
  catch (NotInitializedException or null):
    return  // mediator not ready; skip this tick
  lock (_diffLock):
    added   = current \ running_set
    removed = running_set \ current
    for each (cat, it) in added:
      nina_status_count_started_total.WithLabels(cat, it).Inc()
      nina_status.WithLabels(cat, it).Set(1)
    for each (cat, it) in removed:
      nina_status_count_completed_total.WithLabels(cat, it).Inc()
      nina_status.WithLabels(cat, it).Set(0)
    running_set = current
```

Timer starts in `StartPolling()` from `PrometheusExporterPlugin.Initialize()`. Configurable cadence via `RestartTimer(newSeconds)`.

## 10. Autofocus State Machine (`AutoFocusMetrics.cs`)

Implements `IFocuserConsumer`. Registers via `IFocuserMediator.RegisterConsumer(this)` in `Subscribe()`.

```
state: idle | running

== Transitions ==

idle + AutoFocusRunStarting():
  state = running
  nina_autofocus_running.Set(1)
  if options.AfTimeoutMinutes > 0:
    af_timeout_timer.Start(TimeSpan.FromMinutes(options.AfTimeoutMinutes))

running + UpdateEndAutoFocusRun(AutoFocusInfo info):
  state = idle
  nina_autofocus_running.Set(0)
  af_timeout_timer.Cancel()
  // Spec confirms this fires only on success; no failure signal here

running + BroadcastSuccessfulAutoFocusRun():
  nina_autofocus_success_total.WithLabels(filter).Inc()

running + af_timeout_timer fires:
  state = idle
  nina_autofocus_running.Set(0)
  nina_autofocus_failure_total.WithLabels(reason="timeout").Inc()
```

`IFocuserConsumer.UpdateDeviceInfo(FocuserInfo)` is owned by `FocuserMetrics`, not `AutoFocusMetrics`. Both classes implement `IFocuserConsumer` and register independently; each handles a disjoint set of callbacks (and thus emits a disjoint set of metrics).

Idempotency: duplicate `AutoFocusRunStarting` when already `running` is logged once and ignored (no double-start). Duplicate `UpdateEndAutoFocusRun` while `idle` is ignored.

**JSON FileSystemWatcher** for rsquares metrics:

```
watcher path: %LOCALAPPDATA%\NINA\AutoFocus\
filter: *.json

on Created or Changed event:
  schedule parse 200ms after last event (debounce — handle partial writes)
  parse via System.Text.Json
  extract: filter, method, fitting, backlash_in, backlash_out, rsquares per fit type
  for each fit type ∈ {Quadratic, Hyperbolic, LeftTrend, RightTrend}:
    nina_autofocus_rsquares.WithLabels(filter, method, fitting,
        backlash_in, backlash_out, type=fit_type).Set(rsquare_value)
```

JSON schema reverse-engineered from a real NINA AutoFocus report captured during smoke testing. Field names and structure pinned at parse time; tolerate missing optional fields (NINA versions differ).

## 11. Testing Strategy

xUnit + Moq. Target ~80% line coverage on collector logic. Mediator-coupled code mocked via `I*Mediator` interfaces.

**Test classes** (one per non-trivial logic unit):

| Class | What it covers |
|---|---|
| `SequenceMetricsTests` | Diff: add transitions, remove transitions, simultaneous add+remove, mediator-not-initialized handling, concurrent timer ticks |
| `AutoFocusStateMachineTests` | All transitions in §10, idempotency on duplicate events, timeout fires from `running`, timer cancellation on `UpdateEndAutoFocusRun` |
| `AutoFocusJsonParserTests` | Golden-file test against a captured `%LOCALAPPDATA%\NINA\AutoFocus\*.json`, missing-field tolerance |
| `ImageMetricsTests` | Label derivation from a mocked `ImageSavedEventArgs` — exposure time / filter / gain / offset / binning / HFR / RMS / stats |
| `OptionsChangeHandlerTests` | `PropertyChanged` on Port → `_server.Restart` called with new values; on `SequencePollIntervalSeconds` → `_sequence.RestartTimer` called |
| `MountSideOfPierTests` | Edge counter increments only on transition to `Unknown`, not on repeated `Unknown` reads |

Smoke testing on the developer machine (manual): `make install-dev` → connect simulator devices in NINA → curl `http://localhost:9876/metrics` → diff against the current log-exporter Grafana panels.

## 13. Release / Publishing

Two paths, both per spec section 8:

1. **Manual**: tag `vX.Y.Z` → `make build-package` → GitHub release with zip → users drop into `%LOCALAPPDATA%\NINA\Plugins\Prometheus Exporter\`.
2. **Official plugin manager**: fork `Isbeorn/nina.plugin.manifests` on Bitbucket, add JSON manifest (name, GUID, version, MinimumApplicationVersion, download URL, SHA256, descriptions), submit PR.

Version bump cadence: SemVer. Patch for bug fixes only; minor for new metric families; major if a metric name changes (dashboard-breaking).

## 14. Open Items Carried Into Implementation

1. **GUID generation** — generate once at csproj-creation time via `[guid]::NewGuid()` or `uuidgen`. Frozen forever after (renames break user sequence-saved type references per spec section 3).
2. **Port `9876` final verification** — cross-check Prometheus default port allocations and any obvious NINA-ecosystem conflicts before first publish. Settable via Options anyway, so low-cost to change.
3. **`MinimumApplicationVersion`** — pick a **conservative back-floor**, not the latest NINA release. Users delay upgrades because upgrades carry risk; the floor should be the oldest NINA 3.x where every mediator API referenced by this plugin compiles and runs. Walk backward through `NINA.Plugin` NuGet versions during first build, find the earliest version where all references resolve and smoke tests pass on that NINA build, pin to that version. Bump the floor in a future release only when a new API is genuinely needed.
4. **AutoFocus JSON schema** — exact field names finalized when first real report is captured during smoke test. Parser updated to match.
5. **Default-labels mechanism** — `.WithLabels(profile, host)` per-metric vs. `Metrics.DefaultRegistry` factory wrapper. Picked during implementation; both viable.
